Marapont

Projet scolaire
Description : projet académique de 3 mois réalisé en équipe de 7 personnes, ayant pour objectif de développer une application valorisant le patrimoine physique et culturel français. Le projet met en avant les ponts de France et propose un système de réservation d'activités physiques ou culturelles situées à proximité.

La première phase a été consacrée à l'analyse. Cette étape a permis de choisir un sujet pertinent en lien avec la consigne, d'analyser l'environnement du projet, d'étudier les risques et contraintes, d'identifier les besoins fonctionnels et non fonctionnels, ainsi que de définir les outils et technologies à utiliser.

Lors de la phase de conception, nous avons modélisé les idées de la première phase en plusieurs diagrammes UML. Nous avons également conçu les maquettes et personas.

Durant la phase de développement, j'ai participé au développement des DAO, notamment pour les activités et les groupes d'activités. Côté front-end j'ai fait la page d'accueil et le responsive du site en entier.

Rôles : pendant la phase de cadrage j'ai clarifié l'offre de notre application, identifié les besoins fonctionnels, et également analysé notre environnement en faisant une analyse SWOT et PESTEL. Pendant la phase de conception j'ai réalisé le diagramme des cas d'utilisation, de séquence, participé à la conception du modèle de l'application, puis enfin un BPMN de notre processus métier principal.
TypeScript Astro JS Bun Capacitor Vitest Playwright Docker

Création et analyse d'une BD

Projet scolaire
Description : projet académique en binôme sur la création d'une base de données basée sur le crash du Titanic. La première phase consistait à réaliser le Schéma Entité-Association (SEA) et le Schéma Logique Relationnel (SLR). La deuxième phase a impliqué l'interrogation de la base de données pour répondre à la question : "Quels étaient les facteurs influençant la survie des passagers ?" Le travail a été réparti entre les deux membres du binôme pour les requêtes. Enfin, un document de conclusion avec les requêtes a été remis.

Rôles : pendant la première phase j'ai été en charge de transformer le SEA en SLR, puis pendant la seconde phase il y avait un certain nombre de requêtes à faire nous nous sommes réparti équitablement les requêtes.
SQL PostgreSQL Discord Google Docs

Projet scolaire

Rôles : développement des fonctionnalités principales, intégration du front-end avec Twig, mise en place de la base de données MySQL avec Doctrine ORM, et structuration du projet en MVC.

Mon Carnet de Suivi de Recherche de Stage

Projet scolaire
Description : application complète dédiée au suivi des recherches de stage pour les étudiants IUT. Le système propose une application mobile Android pour les étudiants, une interface web back-office pour les responsables de stage, et centralise la gestion des offres et candidatures via une API sécurisée.

Problème résolu : faciliter et centraliser le processus de recherche de stage en éliminant la dispersion des offres, en simplifiant le suivi des candidatures et en offrant une visibilité claire aux responsables pédagogiques.

Côté architecture du projet, c'est une architecture 4-tiers (Application Mobile + Interface Web + API REST + Base de données) avec pattern MVC respecté sur tous les composants. Modèle de données reconçu vers la 3ème forme normale (3FN) avec 7 entités principales.

La qualité étant au centre de nos préoccupation, mise en place de tests automatisés (Cypress + Espresso), couverture de code >80%, documentation complète (JavaDoc + API Platform), et validation par tests utilisateurs.

Rôles :j'ai participé à la refonte complète de l'application mobile Android dont l'optimisation UX/UI et du cycle de vie des activités. De ce qui est du back-end j'ai fait la mise en conformité avec les améliorations serveur pour les routes API. Lorsqu'ils en manquaient coté android je m'occupé de les codés. Restructuration des classes Activity, et developpement de certaines comme pour les offres d'entreprise dnas la page d'acceuil. Enfin, j'ai fais l'implémentation des tests Espresso

Android Java Symfony API Platform MySQL Docker Espresso Cypress
